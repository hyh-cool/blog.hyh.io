# 一、介绍

致谢

Gungnir v2 是一个基于 Vuepress 2 的博客主题，不管是UI还是功能都很符合我的心意，于是经过几天的照搬照抄🥲🥲🥲，终于将我的博客搭建起来了。在此感谢Xiaohan Zou这位大佬开发了这款主题🤞🤞🤞！

# 二、搭建指南

由于个人从事的是嵌入式开发领域，对Vue.js这款框架以及CSS、JavaScript语法均不熟悉，在这仅仅记录下我的搭建过程，如果你也跟我一样🥲🥲🥲，可以在我的github仓库下载源码进行搭建。

下载后在根目录下执行以下命令：

安装依赖

yarn
1

在本地启动服务器，VuePress 会在 http://localhost:8080启动一个热重载的开发服务器。

yarn dev
1

生成静态文件

yarn build
1

如果你跟我一样使用Github Pages部署的话，我的源码根目录下包含了GitHub Actions配置文件（在.github/workflows 目录），可以自动部署，你只需将你的源码推送到你的github个人博客仓库中。

GitHub Actions配置文件详情
name: build and deploy VuePress to GitHub Pages automatically

on:
  push:
    branches: [ master ]
  pull_request:
    branches: [ master ]

jobs:
  build-and-deploy-vuepress:
    runs-on: ubuntu-latest

    steps:
      # Checks-out your repository under $GITHUB_WORKSPACE, so your job can access it
      - name: Checkout
        uses: actions/checkout@v2

      # Setup Node.js
      - name: Setup Node
        uses: actions/setup-node@v2.1.2
        with:
          node-version: '12'

      - name: Get yarn cache
        id: yarn-cache
        run: echo "::set-output name=dir::$(yarn cache dir)"

      - name: Cache dependencies
        uses: actions/cache@v2
        with:
          path: ${{ steps.yarn-cache.outputs.dir }}
          key: ${{ runner.os }}-yarn-${{ hashFiles('**/yarn.lock') }}
          restore-keys: |
            ${{ runner.os }}-yarn-

      # Build VuePress
      - name: Build
        run: |
          yarn install --frozen-lockfile
          yarn build

      # Push it to gh-pahes
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: dist
          # 注意：下面填你自己的Github Pages 站点别名！！！
          cname: hyh.cool

我的目录结构如下：

.
├── blog
│   ├── .vuepress //用于存放全局的配置、组件、静态资源等
│   │   ├── components //该目录中的 Vue 组件将会被自动注册为全局组件。
│   │   │   └── About.vue //关于页配置，把里面的github链接和邮箱改成你自己的
│   │   ├── public //静态资源目录。
│   │   ├── styles //用于存放样式相关的文件。
│   │   ├── config.js //配置文件的入口文件，把里面信息替换成你自己的就可以了。
│   │   └── enhanceApp.js //客户端应用的增强，主要是引入了一些图标
│   ├── about  //关于页
│   ├── docs   //文档链接页
│   ├── LeetCode //存放数据结构与算法相关文章
│   ├── NetWork  //存放计算机与网络相关文章
│   ├── OS       //存放操作系统相关文章
│   ├── posts    //存放首页文章
└── package.json
